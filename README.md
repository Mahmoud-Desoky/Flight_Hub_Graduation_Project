Introduction### 1.1 PurposeThis test plan documents the comprehensive quality assurance testing performed across three distinct systems: the **FlightHub** travel booking website (UI automation), the **PetStore REST API** (API testing), and an **E-Commerce application database** (database testing). The plan outlines test cases executed, defects discovered, and recommendations for remediation.The document serves as a formal record of testing activities, results, and identified issues for stakeholders, developers, and the QA team.### 1.2 ScopeTesting coverage includes:- **FlightHub Website**: User authentication (login/registration) and flight search functionality- **PetStore API**: Store, Pet, and User endpoint validation- **E-Commerce Database**: Data integrity, constraint validation, and CRUD operations across multiple tables### 1.3 Test Environment| Component | Details ||-----------|---------|| **FlightHub UI** | Java + Selenium WebDriver + TestNG + Maven || **PetStore API** | REST API endpoints (Swagger PetStore) || **E-Commerce DB** | SQL Database with 6 core tables || **Test Approach** | Automated UI + Manual Exploratory + API + Database |---## 2. DefectsDuring manual exploratory testing of the FlightHub website, the following significant defects were discovered that were not covered by the automated test suites:#### Defect 1: Registration Name Validation Bypass| Field | Value ||-------|-------|| **Severity** | Medium || **Status** | **OPEN** |**Description:** The registration form accepts numeric values, special characters, and excessively long strings in the First Name and Last Name fields without validation. The system should enforce alphabetic constraints and reasonable length limits for name fields.**Reproduction Steps:**1. Navigate to the registration form2. Enter `123456789` in the First Name field3. Enter `!@#$%^&*()` in the Last Name field4. Complete registration — system accepts invalid names| Expected Result | Actual Result ||-----------------|---------------|| Validation error rejecting numeric-only and special-character names | Registration completes successfully with invalid name data |#### Defect 2: Flight Search Missing Origin Validation| Field | Value ||-------|-------|| **Severity** | High || **Status** | **OPEN** |**Description:** The flight search functionality allows users to submit searches without specifying an origin airport. This bypasses a critical validation check and produces meaningless search results or errors.| Expected Result | Actual Result ||-----------------|---------------|| Validation modal preventing search without origin | Search submits with empty origin field |#### Defect 3: Missing Arabic Countries in Billing Information| Field | Value ||-------|-------|| **Severity** | High || **Status** | **OPEN** |**Description:** During the booking phase, the billing information country dropdown contains only UAE as an Arabic country option. However, users can search for and book flights originating from Egypt, Saudi Arabia, and other Arabic countries. This creates a critical blocker where travelers from these countries cannot complete their booking because their country is unavailable in the billing address form.| Expected Result | Actual Result ||-----------------|---------------|| All countries with available flights should appear in the billing country dropdown | Only UAE is listed among Arabic countries, preventing booking completion for passengers from Egypt and other Arabic nations |---## 3. Summary & Recommendations### 5.1 Test Execution Summary| Test Area | Total Tests | Passed | Failed | Pass Rate ||-----------|:-----------:|:------:|:------:|:---------:|| **FlightHub UI** (Automated) | 30+ | Automated | N/A | N/A || **FlightHub** (Manual Findings) | 3 defects | — | 3 open | — || **PetStore API** | 10 | 8 | 2 | **80%** || **E-Commerce DB** | 18 | 14 | 4 | **78%** |### 5.2 Key Findings- **FlightHub:** 3 critical manual defects identified beyond automated coverage — name validation bypass, missing origin validation, and incomplete country support for Arabic markets.- **PetStore API:** 2 of 10 API tests failed (20% failure rate) — pet retrieval by ID and user lookup by username endpoints returned unexpected responses.- **E-Commerce DB:** 4 of 18 database tests failed (22% failure rate) — missing primary key constraints, negative price acceptance, NULL foreign key insertion, and invalid product reference insertion indicate inadequate database-level validation.### 5.3 Recommendations| # | Recommendation | Priority ||---|----------------|----------|| 1 | **FlightHub Registration:** Implement regex validation for First/Last Name fields (alphabetic characters only, max 50 characters). Add front-end and back-end validation layers. | Medium || 2 | **FlightHub Search:** Add mandatory origin field validation before search submission. Display clear error messaging when required fields are empty. | High || 3 | **FlightHub Billing:** Expand billing country dropdown to include all countries available in the flight search origin/destination lists. Ensure parity between flight availability and billing address support. | High || 4 | **PetStore API:** Review pet retrieval and user lookup endpoint implementations. Verify database seeding and ID generation consistency. | High || 5 | **E-Commerce Database:** Add PRIMARY KEY constraints to `user_id`. Implement CHECK constraints for `price &gt;= 0`, `rating &lt;= 5`, and NOT NULL foreign key references. Add foreign key validation with ON DELETE/UPDATE rules. | Critical |---&gt; **Document Version:** May 2026  &gt; **Author:** QA Team  &gt; **Status:** Final Report---## Appendix: Quick Reference&lt;details&gt;&lt;summary&gt;&lt;b&gt;Click to expand: Defect Severity Legend&lt;/b&gt;&lt;/summary&gt;| Severity | Description ||----------|-------------|| **Critical** | Blocks core functionality; no workaround || **High** | Major functionality impacted; limited workaround || **Medium** | Moderate impact; workaround available || **Low** | Minor issue; cosmetic or edge case |&lt;/details&gt;&lt;details&gt;&lt;summary&gt;&lt;b&gt;Click to expand: Test ID Naming Convention&lt;/b&gt;&lt;/summary&gt;| Prefix | Module ||--------|--------|| `TC-LG` | Login || `TC-RG` | Registration || `TC-FS` | Flight Search || `TC-PS` | PetStore API || `FGG3` | Failed Test Tracker |&lt;/details&gt;
